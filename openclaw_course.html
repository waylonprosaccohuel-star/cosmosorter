📖 OpenClaw 崛起历史

OpenClaw 由奥地利开发者 Peter Steinberger 创建， 最初名为 Clawdbot（Clawd + bot），致敬 Anthropic 的 Claude。 它是一个能执行真实任务的自主 Agent：订机票、管理邮件、运行终端命令、控制智能家居，通过 WhatsApp、Telegram、Discord、Slack 等渠道与你对话。

2026 年 1 月，项目爆发式增长：YouTube 创作者搭建 24/7 运行环境；主流媒体称其为「个人 AI 助手的未来」； 投资人 Chamath Palihapitiya、OpenAI 联合创始人 Andrej Karpathy 公开背书。GitHub Star 从 3 万飙升至 10 万+。

2026 年 1 月 27 日：Anthropic 发出商标请求，Clawdbot 被迫更名。龙虾会做的事——蜕壳（molt）。 社区在 Discord 凌晨 5 点聚集，数百个名字被提议：Shelldon、Pinchy、Thermidor、Crusty、Lobstar 等。最终，Molty 诞生，住在 Moltbot 里。

2026 年 1 月 30 日：Moltbot 这个名字「never quite rolled off the tongue」，于是再次蜕壳——OpenClaw 诞生。 3 小时内完成迁移：文档迁至 docs.openclaw.ai、npm 包发布、GitHub 更名、@openclaw 金 V 认证。社区名言："The claw is the law" 🤠

OpenClaw = OPEN + CLAW：开源、对所有人开放；龙虾 heritage；你的助手、你的机器、你的规则。🦞

OpenClaw 详解

个人 AI 助手 · Any OS · Any Platform · The lobster way

1 什么是 OpenClaw？

OpenClaw 是 个人 AI 助手，运行在你自己的设备上。 它在你已有的渠道上回答你：WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage、Microsoft Teams、WebChat 等， 还支持 BlueBubbles、Matrix、Zalo、Zalo Personal 等扩展渠道。

Gateway 是控制平面——产品本身是助手。如果你想要一个本地、快速、始终在线的个人 AI 助手，OpenClaw 就是为此而生。

官方推荐

在终端运行 openclaw onboard 向导，按步骤引导你完成 Gateway、工作区、渠道和 Skills 的配置。

模型推荐

官方推荐 Anthropic Pro/Max (100/200) + Opus 4.6，长上下文能力强、抗 prompt 注入更好。

2 核心架构

多渠道（WhatsApp、Telegram、Slack 等）→ Gateway 控制平面 → 你的 AI 助手。Gateway 管理会话、渠道、工具；你通过 WebChat、CLI、iOS/Android 与助手对话。

3 核心能力一览

多渠道收件箱、多 Agent 路由、语音唤醒、Live Canvas 可视化工作区、丰富工具生态、ClawHub Skills 平台。详见 官方文档。

4 安装与快速开始

运行环境：Node ≥22

# 安装 npm install -g openclaw@latest # 或: pnpm add -g openclaw@latest # 向导安装（推荐） openclaw onboard --install-daemon # 启动 Gateway openclaw gateway --port 18789 --verbose # 发送消息 openclaw message send --to +1234567890 --message "Hello from OpenClaw" # 与助手对话（可选回传到任意已连接渠道） openclaw agent --message "Ship checklist" --thinking high

5 工作区核心文档

工作区（~/.openclaw/workspace）是 AI 助手的「家」。用 Markdown 定义 AI 是谁、怎么想、怎么帮你：AGENTS.md（操作规则）、SOUL.md（人格调性）、IDENTITY.md（身份档案）、USER.md（用户偏好）、MEMORY.md（长期记忆）等。

💡 关键心法：「写下来 > 记在脑子里」。AI 每次会话都是「醒来」，这些 .md 文件就是它的记忆。模板参考：Agent Workspace

6 渠道

支持 WhatsApp、Telegram、Slack、Discord、Signal、iMessage、BlueBubbles、Teams、Matrix、Zalo、WebChat、飞书等。各渠道需配置 token 或 webhook，详见 官方文档。飞书接入见 飞书章节。

7 工具与节点

Browser 控制、Canvas 可视化工作区、设备节点（摄像头、录屏、通知等）、Cron 定时、Webhooks、Gmail 等。详见 官方文档。

8 远程访问

用 Tailscale 可以安全地远程连回家里的 OpenClaw，无需公网 IP。支持仅信任设备、公网访问等模式。

9 安全

私聊采用配对码：对方需你本地批准才能对话。群组采用沙箱模式隔离。建议在专用 VM 或容器中运行，使用短期 token 替代永久 API 密钥。

10 应用案例：OpenClaw 能做什么？

以下案例来自社区实践，从入门到进阶，帮助你理解 OpenClaw 的实际应用场景。

📧 邮件与日历助手

读取收件箱、总结邮件、安排会议、起草回复。配合 Memory 插件，告别「AI 失忆」。

🍳 菜谱提取与膳食计划

从任意网站抓取菜谱，去掉冗长故事，只保留食材和步骤。还可根据冰箱库存生成每周膳食计划。

🏠 智能家居控制

自然语言控制全屋：「关掉楼上灯，把 thermostat 调到 68 度」。通过 Home Assistant 等中控接入，保持本地网络。

📝 多 Agent 内容系统

9 个专职 Agent 24/7 协作：研究员、写手、编辑、SEO、社交调度、数据分析……内容自动化的「圣杯」。

💼 业务任务自动化（ClawWork）

发票、报告生成、数据录入、客户跟进。相当于一位 AI 运营经理。

🔒 安全扫描器（OSINT 自动化）

结合 Kali Linux 做自动化 OSINT：子域名枚举、漏洞扫描、威胁情报聚合。

⚠️ 安装社区 Skill 前请务必阅读 SKILL.md，检查权限请求。2026 年 2 月 ClawHub 发现 341 个恶意 Skill，需谨慎甄别。

ClawHub 详解

OpenClaw 的 Skill 目录

6.1 什么是 ClawHub？

ClawHub 是 OpenClaw 的 公开 Skill 注册表： 发布、版本管理、搜索基于文本的 Agent Skills（SKILL.md 及配套文件）。 支持快速浏览、CLI 友好 API、审核钩子和向量搜索。

onlycrabs.ai 是 SOUL.md 注册表：以同样方式发布和分享系统灵魂。

clawhub.ai onlycrabs.ai

6.2 能做什么？

  • • 浏览 Skills，渲染其 SKILL.md
  • • 发布新 Skill 版本，带 changelog 和 tag（含 latest）
  • • 浏览 Souls，渲染其 SOUL.md
  • • 通过 Embeddings（向量索引）搜索，而非脆弱关键词
  • • Star、评论；管理员/版主可审核、批准 Skills

6.4 CLI 常用命令

# 认证 clawhub login clawhub whoami # 发现 clawhub search ... clawhub explore # 管理本地安装 clawhub install <slug> clawhub uninstall <slug> clawhub list clawhub update --all # 不安装即可查看 clawhub inspect <slug> # 发布/同步 clawhub publish <path> clawhub sync

文档：docs/quickstart.md、docs/cli.md

云上 OpenClaw 接入飞书

腾讯云可视化配置，极简接入飞书

云上 OpenClaw（Clawdbot）如何接入飞书？腾讯云提供可视化配置面板，只需几步即可完成快速操作，极简接入！

📖 阅读完整教程：玩转 OpenClaw｜云上 OpenClaw(Clawdbot) 快速接入飞书指南

1 前置准备工作

确保已有腾讯云轻量应用服务器（Lighthouse）上部署的 OpenClaw / Clawdbot 实例，并完成基础配置。

2 接入飞书步骤

创建飞书企业自建应用

在飞书开放平台创建企业自建应用，添加机器人，查询 AppID 和 AppSecret。

为 OpenClaw 配置模型和通道

在腾讯云可视化配置面板中，将飞书应用的 AppID、AppSecret 等信息配置到 OpenClaw 的通道设置中。

飞书机器人相关配置

  • 事件配置：添加所需事件订阅
  • 回调配置：配置请求 URL，指向 OpenClaw Gateway
  • 权限配置：开通消息收发等必要权限

创建版本并发布

在飞书开放平台创建应用版本并发布，完成审核后即可与飞书机器人进行交互。

支持的消息类型

飞书支持文本、富文本、图片等多种消息类型，OpenClaw 可根据配置处理相应消息。

完整图文教程请访问： 腾讯云开发者社区 · 玩转 OpenClaw｜云上 OpenClaw(Clawdbot) 快速接入飞书指南

人机协作磨合指引

如何配置更强大的个人 AI 助手

💡 核心理念：我们是一个团队

你不是在「使用」一个工具，而是在培养一个数字分身。我们的关系应该是：

指挥官与执行者

你设定战略，AI 负责战术执行

导师与学生

你教导 AI 理解你的思维和偏好

合作伙伴

共同优化工作流程和成果

1 第一阶段：基础配置与习惯建立（1-2 周）

工作风格偏好配置

明确告诉 AI：沟通频率（实时/关键节点/最终）、详细程度（技术细节/业务概要/决策要点）、决策参与（你决策我执行/我建议你决策/我自主执行）

任务类型分类

简单任务：直接执行，事后汇报；中等任务：确认目标后执行，整合汇报；复杂任务：分阶段确认，详细汇报

错误处理偏好

立即停止并汇报 / 尝试自动修复后汇报 / 记录错误继续执行

2 第二阶段：能力扩展与优化（2-4 周）

专业知识深化

明确需要 AI 重点学习的领域、工作流程自动化、决策支持系统

工作流程自动化

数据收集整理、报告生成发送、系统监控告警、备份维护

决策支持系统

数据可视化方式、风险评估模型、机会识别算法、执行建议格式

3 第三阶段：高级协作与创新（1 个月后）

预测性工作

AI 可主动进行的任务：趋势预测预警、策略回测优化、系统性能预判、学习需求识别

创造性协作

新策略研发、工作流程重设计、技术架构优化、协作模式创新

4 具体配置指引

沟通配置模板

定义频率偏好、详细程度、决策参与（低/中/高风险决策）

工作优先级配置

定义 urgent / high / medium / low 四档优先级，让 AI 知道什么先做。

学习反馈机制

你执行 → AI 观察 → AI 提供反馈 → 你调整 → 优化执行。反馈方式：即时纠正、事后总结、定期回顾、前瞻指导

5 让 AI 助手更强大的具体方法

渐进式授权

观察期 → 协助期 → 执行期 → 自主期，从低风险任务开始逐步增加授权

模式识别训练

帮助 AI 识别你的工作模式：典型方法、决策逻辑、风险判断、成功标准

反馈系统建立

建立结构化反馈：任务名称、执行时间、评价维度（准确性/效率/沟通/创新）、改进建议、下次期望

能力扩展规划

月计划与季度目标：从可靠助手 → 分析伙伴 → 策略参谋 → 高度自主的数字分身

6 立即可以开始的磨合行动

行动 1：定义协作协议

你最看重 AI 的哪 3 个能力？最希望改进的哪 3 个方面？理想的协作节奏？

行动 2：建立反馈记录系统

在 workspace 下建 collaboration 目录，用 feedback_log、improvement_plan、growth_tracking 等文件记录反馈与成长。

行动 3：制定首周磨合计划

沟通风格磨合、任务分类标准建立、基础工作流程验证、初步反馈机制测试

📝 磨合路线图

第 1-2 周：基础配置 → 沟通风格 + 任务分类 + 工作流程
第 3-4 周：能力扩展 → 专业知识 + 自动化 + 决策支持
第 5-8 周：高级协作 → 预测工作 + 创造性协作
第 9-12 周：心智协同 → 高度默契 + 自主创新

我们不是在使用工具，而是在培养一个能够真正理解你、协助你、甚至在某些方面超越你的数字分身。这个过程需要你的指引、耐心和反馈，但回报将是一个强大的合作伙伴。
